# GNUClient Scripting API

Runtime Java scripting for GNUClient. Drop a `.java` file in the scripts
folder, click **Reload Scripts** in the ClickGUI's Scripts tab, and it
compiles, loads, and registers as a live module — no game relaunch, no
Gradle rebuild.

This document is the ground-truth reference for the API surface as it
exists in `client/src/main/java/gnu/client/script/`. It was generated by
reading the actual committed source, not from memory — if this file and
the code ever disagree, the code wins; please update this doc.

## Where scripts live

```
~/.config/gnuclient/scripts/*.java
```

Created automatically on first load if missing. Compiled output goes to
`~/.config/gnuclient/scripts_compiled/` (safe to delete; regenerated on
reload).

## Reloading

Open the ClickGUI, go to the **Scripts** tab, click **Reload Scripts**.
This unregisters + closes every currently loaded script, re-scans the
scripts folder, recompiles and reloads everything, then re-runs
`ConfigManager.load()` so saved script settings are restored.

> There is also an `RShift+R` debug keybind intended to trigger the same
> reload (`ClientEventListener.checkScriptReloadCombo()`), but as of this
> writing it has never been confirmed to actually fire — the button is the
> reliable path. Worth investigating if you have time; not blocking.

Toggling a script on/off uses the **same mechanism as any built-in
module** — click its row in the ClickGUI (or bind a key to it). Scripts do
not need their own "Enabled" setting; `onPreUpdate()` is only called while
the module itself is enabled.

## Script file syntax

A script file is **not** a standalone compilable Java file — do not
include `package`, `import`, or a `class` declaration. Just write bare
field declarations and bare methods, as if writing the body of a class
that already exists.

> **Do not use `import` statements.** The script compiler wraps your body
> into a generated class that already imports `java.util.List` and
> `java.util.ArrayList`. Extra imports cause compile errors.

```java
// RenamedScript.java
float multiplier = 1.3f;

void onLoad() {
    modules.registerSlider("Multiplier", 1.3f, 1.0f, 3.0f);
}

void onPreUpdate() {
    if (!keybinds.isMovementDown()) return;
    float mult = modules.getSlider("Multiplier");
    double mx = client.getMotionX();
    double mz = client.getMotionZ();
    double speed = Math.sqrt(mx * mx + mz * mz);
    if (speed > 0.001) {
        double target = 0.2873 * mult;
        double scale = target / speed;
        client.setMotion(mx * scale, client.getMotionY(), mz * scale);
    }
}
```

Under the hood, `ScriptManager` wraps this body into a generated class:

```java
package gnu.client.script.generated;
// ... imports ...
public final class sc_RenamedScript_a1b2c ends Module implements PacketListener {
    public static final String scriptName = "RenamedScript";
    public static final Client client = Client.INSTANCE;
    public static final World world = World.INSTANCE;
    public static final Keybinds keybinds = Keybinds.INSTANCE;
    public static final Inventory inventory = Inventory.INSTANCE;
    public static final Packets packets = Packets.INSTANCE;
    public static final Status status = Status.INSTANCE;
    public static final Util util = Util.INSTANCE;
    private final Modules modules;
    // ... your pasted body goes here ...
}
```

Fields declared in your script are real instance fields on this generated
class and persist across `onPreUpdate()` calls exactly like a normal
module. The class name (`sc_<name>_<random>`) is regenerated on every
reload, so don't rely on it.

## Lifecycle hooks

All optional (a script with none of these is legal but does nothing).

| Method | Signature | Called | Notes |
|---|---|---|---|
| `onLoad` | `void onLoad()` | Once, immediately after registration, **before** `ConfigManager.load()` runs | Register settings here via `modules.registerButton`/`registerSlider` |
| `onPreUpdate` | `void onPreUpdate()` | Once per tick at **ClientTick START** | Main tick logic |
| `onPostUpdate` | `void onPostUpdate()` | Once per tick at **ClientTick END** (optional) | End-of-tick logic |
| `onScriptDisable` | `void onScriptDisable()` | When the module is disabled | **Not** `onDisable` — that name collides with `Module`'s own override. Use `onScriptDisable`. |

If any of these throws, the error is logged (`GnuLog`, visible in
`/tmp/gnu_debug.log`) and the script's module is automatically disabled
(`setEnabled(false)`) — except `onScriptDisable` itself, which won't
re-trigger the disable to avoid a loop.

## Packet hooks (optional)

Declare either or both to opt a script into packet interception. **A
script that declares neither of these never registers as a
`PacketListener` at all** — zero dispatch overhead for non-network
scripts.

| Method | Signature | Thread | Notes |
|---|---|---|---|
| `onPacketSend` | `boolean onPacketSend(Object packet)` | Main client thread | Return `true` to cancel the send, `false` to pass through |
| `onPacketReceive` | `boolean onPacketReceive(Object packet)` | **Netty channel thread — NOT the main client thread** | Return `true` to cancel/consume, `false` to pass through |
| `packetSendPriority` | `int packetSendPriority()` | — | Optional. Defaults to **`-100`** if not declared. |

**Default priority is intentionally very low (`-100`).** GNUClient's
native `PacketListener`s are ordered (`FreeLook=200 > AutoBlock=100 >
Lagrange/KnockbackDelay/Backtrack=0`). A script defaulting below all of
them means a careless packet script can never accidentally steal a packet
away from an already-tuned native lag module. Only raise
`packetSendPriority()` above `0` if you specifically need to contend with
those modules for the same packets, and understand what you're
overriding.

**Thread safety warning:** `onPacketReceive` fires on the Netty thread.
Any field also read/written from `onPreUpdate()` (main thread) is shared
mutable state across threads with no automatic synchronization. If you
need to communicate from `onPacketReceive` to `onPreUpdate`, use an
`AtomicInteger`/`AtomicBoolean`/`volatile` field, not a plain field.

If a packet hook throws, the script is auto-disabled, same as tick hooks.

### Example — blink

```java
List<Object> buffer = new ArrayList<Object>();
long bufferStart = 0;

void onLoad() {
    modules.registerSlider("Delay", 200f, 50f, 1000f);
}

boolean onPacketSend(Object packet) {
    if (!packets.isMovement(packet)) return false;
    if (bufferStart == 0) bufferStart = client.time();
    buffer.add(packet);
    return true; // cancel — packet is held, not dropped
}

void onPreUpdate() {
    if (buffer.isEmpty()) return;
    long delay = (long) modules.getSlider("Delay");
    if (client.time() - bufferStart >= delay) {
        for (Object p : buffer) packets.sendReleased(p);
        buffer.clear();
        bufferStart = 0;
    }
}

void onScriptDisable() {
    // flush anything still buffered so disabling doesn't desync you permanently
    for (Object p : buffer) packets.sendReleased(p);
    buffer.clear();
}
```

## Implicit accessors

These are available with **no import statements** — they're static fields
injected by the generated wrapper.

### `client` — player state, rotation, motion, timer

```java
Object getPlayer()                          // raw EntityPlayerSP, or null if not in-game
long   time()                               // System.currentTimeMillis()
Object raycastBlock(double distance, float yaw, float pitch)  // MovingObjectPosition, or null on miss

double getMotionX()
double getMotionY()
double getMotionZ()
void   setMotion(double x, double y, double z)

float  getYaw()
float  getPitch()
void   setRotation(float yaw, float pitch)  // pitch clamped to [-90, 90] internally

boolean isOnGround()
boolean isSneaking()
boolean isSprinting()

float  getTimerSpeed()
void   setTimerSpeed(float speed)           // UNCLAMPED — any value is passed through
void   resetTimer()                         // reset to vanilla (1.0)

double getPosX() / getPosY() / getPosZ()  // local player position
void   setJump(boolean jump)                // vanilla jump input

boolean isRiding()
Object  getRidingEntity()                   // Entity being ridden, or null
void    setRidingMotion(double x, double y, double z)
double  entityPosX/Y/Z(Object entity)
void    setEntityPosition(Object entity, double x, double y, double z)
void    setEntityVelocity(Object entity, double x, double y, double z)
void    setEntityYaw(Object entity, float yaw)
void    sendSteer(float strafe, float forward, boolean jump, boolean unmount)
                                            // C0C steer — keep |strafe|, |forward| <= 0.98 (Grim VehicleA)
```

> Note: `setRotation` does not affect FreeLook's camera path — FreeLook
> uses its own separate look-direction state, not the real player
> rotation. If FreeLook is enabled, `setRotation` still writes the real
> player rotation (affecting the server-visible facing) but you won't see
> the change in your own camera.

### `world` — block and entity queries

```java
Object   getBlockAt(int x, int y, int z)    // IBlockState, or null if no world
List<?>  getEntities()                       // all loaded entities, empty list if no world
```

### `keybinds` — mouse and movement key state

```java
boolean isMouseDown(int button)   // 0 = LMB (native evdev), 1 = RMB (LWJGL). anything else = false
boolean isForwardDown()
boolean isBackDown()
boolean isLeftDown()
boolean isRightDown()
boolean isMovementDown()          // true if ANY of the above four is held
boolean rightClick()              // synthesize a vanilla right-click / item-use; returns true if server accepted it
```

There is no `leftClick()` — only `isMouseDown(0)` to read state, no
synthetic left-click exists yet.

### `inventory` — hotbar and slots

```java
Object getStackInSlot(int slot)   // ItemStack for slot 0-35 (0-8 = hotbar), or null if empty/unavailable
int    getSlot()                  // current hotbar index 0-8, or -1 if unavailable
```

### `util` — randomness

```java
int    randomInt(int min, int max)      // inclusive [min, max]
double randomDouble(double min, double max)  // [min, max)
```

### `modules` — per-script settings (instance, not static)

Unlike the other accessors, `modules` is constructed fresh per script
instance (it needs to know which `Module` it's attached to), so it's not
a shared singleton — but you use it exactly the same way.

```java
void    registerButton(String name, boolean defaultValue)
void    registerSlider(String name, float defaultValue, float min, float max)
boolean getButton(String name)     // false if never registered or wrong type
float   getSlider(String name)     // 0f if never registered or wrong type
String  getScriptName()
```

Settings registered this way are **real** `BoolSetting`/`SliderSetting`
objects attached to the script's module — they render in the ClickGUI
exactly like a hand-written module's settings, and persist through
`ConfigManager`'s save/load just the same.

### `packets` — packet inspection and mutation

Only meaningful inside `onPacketSend`/`onPacketReceive`, though nothing
stops you calling the predicate methods elsewhere with a packet object you
obtained some other way.

```java
boolean isAttack(Object packet)
boolean isUseEntity(Object packet)
boolean isMovement(Object packet)          // C03-family movement packet
boolean hasPosition(Object packet)
boolean isAnimation(Object packet)
boolean isBlockPlacement(Object packet)
boolean isReleaseUseItem(Object packet)
boolean isKeepAlive(Object packet)
boolean isTransaction(Object packet)       // client OR server confirm-transaction
boolean isVelocity(Object packet)          // S12, any entity
boolean isSelfVelocity(Object packet)      // S12, self only
boolean isExplosion(Object packet)
boolean isPlayerPosLook(Object packet)

String  simpleName(Object packet)          // e.g. "C03PacketPlayer" — for logging/debugging
int     entityId(Object packet)            // -1 if not applicable

double  movementX(Object packet)
double  movementY(Object packet)
double  movementZ(Object packet)
boolean movementOnGround(Object packet)

int     velocityMotionX(Object packet)
int     velocityMotionY(Object packet)
int     velocityMotionZ(Object packet)

void    setMovementRotation(Object packet, float yaw, float pitch)
                                            // rewrites yaw/pitch on a C03 packet in place
                                            // (no-op if the packet isn't a movement packet)

void    sendReleased(Object packet)        // send a previously-cancelled packet (blink release)
void    processInbound(Object packet)      // hand a packet to the vanilla NetworkManager
```

There is no generic field-mutation/reflection access to arbitrary packet
fields — the surface above is intentionally narrow (matching what
disablers and simple network modules actually need) rather than exposing
raw packet internals to scripts.

### `status` — combat and entity state

Every method has both a no-arg (local player) and an `(Object entity)`
overload unless noted.

```java
boolean isUsingItem() / isUsingItem(Object player)
boolean isBlocking()  / isBlocking(Object player)

int     getHurtTime()      / getHurtTime(Object entity)
int     getMaxHurtTime(Object entity)      // no no-arg overload
int     getDeathTime(Object entity)        // no no-arg overload

float   getHealth()      / getHealth(Object entity)
float   getMaxHealth()   / getMaxHealth(Object entity)
float   getAbsorption()  / getAbsorption(Object entity)

boolean isDead()  / isDead(Object entity)
boolean isAlive() / isAlive(Object entity)

boolean isSwingInProgress() / isSwingInProgress(Object player)

Object  getHeldItemStack() / getHeldItemStack(Object player)
boolean isHoldingSword()     // no entity overload
boolean isHoldingBlock()     // no entity overload

boolean isInWater() / isInWater(Object entity)
```

**Deliberately not included** (flagged during design as unverified — do
not add without confirming the real SRG field/method against a mapping
reference first):
- `isInLava()`
- `isCollidedHorizontally()`
- `canAttackNow()` (1.8.9 has no attack-cooldown system; this was a
  speculative guess, never confirmed against a real call site)

## Design notes for anyone extending this API

- **Every accessor facade is a stateless singleton** (`Client.INSTANCE`,
  `World.INSTANCE`, etc.) except `Modules`, which is constructed per
  script because it needs a reference to its owning `Module`. If you add
  a new facade, do not cache game objects (player/world/entity
  references) as fields — resolve through `McAccess` fresh on every call.
  A cached reference pins that generated script's classloader in memory
  even after the script is reloaded/removed.
- New accessors should wrap an **existing** `McAccess`/`PacketHelper`/
  `PacketUtil` method wherever one already exists, rather than writing
  fresh reflection. If no such method exists, add it to `McAccess`
  first (matching its existing `invoke(gameClass(...), "func_...", ...)`
  pattern), then wrap it here — don't reflect directly from a script
  facade.
- The classloader parent for loaded scripts is
  `ScriptManager.class.getClassLoader()`, **not**
  `McAccess.gameLoader()`. This guarantees the script's `Module`
  superclass is the exact same `Class` object `ModuleManager` already
  has loaded. Using the wrong loader here causes a `ClassCastException`
  on every script load — see `McAccess.java`'s header comment on the
  observed dual-classloader case for why this matters.
- The compiler's classpath is resolved by walking the classloader chain
  for a `URLClassLoader` whose `getURLs()` contains `gnu-client.jar` —
  **not** `getProtectionDomain().getCodeSource()`, which is unreliable
  under `LaunchClassLoader` (the jar is added via `addURL()` at runtime,
  so it's never on the system classpath property). See
  `ScriptManager.ourJarPath()`.
